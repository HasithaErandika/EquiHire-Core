# EquiHire Frontend

## Overview

This repository contains the frontend application for EquiHire, an AI-Native Blind Assessment Platform. The application is built using React, TypeScript, and Vite, providing a fast, responsive, and secure interface for both candidates and recruiters.

The frontend is designed to facilitate a "Bias-Free" recruitment process by providing anonymized interfaces and secure assessment environments.

---

## Core Features

### Candidate Experience
*   **Secure Assessment Interface:** A distraction-free environment for completing technical exams.
*   **Anonymized Profile Management:** Candidate data is handled with privacy in mind, focusing on skills rather than personal identifiers.
*   **Interactive Growth Reports:** Visualized feedback and performance metrics generated by AI.

### Recruiter Dashboard
*   **Bias-Filtered View:** Access to candidate scores and technical evaluations without exposure to PII.
*   **Assessment Configuration:** Tools to define and manage recruitment campaigns and technical rubrics.
*   **Analytics Engine:** High-level metrics on candidate performance and recruitment efficiency.

---

## Technical Stack

### Core Frameworks
*   **React:** For building modular and high-performance UI components.
*   **Vite:** Optimized build tool for rapid development and production bundling.
*   **TypeScript:** Static typing for enhanced code quality and developer productivity.

### Styling and Animation
*   **Tailwind CSS:** Utility-first CSS framework for responsive design.
*   **Framer Motion:** High-performance animation library for smooth transitions and interactions.
*   **Radix UI:** Unstyled, accessible primitives for building high-quality design systems.
*   **Lucide React:** Consistent and modern icon set.

### Integration and State
*   **Asgardeo Auth React:** Seamless integration with WSO2 Asgardeo for secure identity management.
*   **Supabase JS:** Client-side integration for data persistence and real-time updates.
*   **React Hook Form + Zod:** For robust form management and schema-based validation.
*   **Recharts:** Data visualization for candidate feedback and dashboards.

## Getting Started

### Prerequisites
*   Node.js (LTS version)
*   npm or yarn

### Installation
1.  Navigate to the directory:
    ```bash
    cd react-frontend
    ```
2.  Install dependencies:
    ```bash
    npm install
    ```

### Development
Start the development server with Hot Module Replacement (HMR):
```bash
npm run dev
```

### Building for Production
Create an optimized production build:
```bash
npm run build
```

### Running Tests
Execute the test suite using Vitest:
```bash
npm run test
```

## Environment Configuration

The application requires specific environment variables to function correctly. Copy the example file and populate it with your credentials:

```bash
cp .env.example .env
```

Key variables include:
*   `VITE_ASGARDEO_CLIENT_ID`: OAuth2 client ID for identity management.
*   `VITE_ASGARDEO_BASE_URL`: The base URL of your Asgardeo organization.
*   `VITE_API_BASE_URL`: The endpoint for the Ballerina backend gateway.
*   `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY`: Connection details for data persistence.
